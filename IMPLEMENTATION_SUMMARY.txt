WORKFLOW STATES
================================================================================

COMPLETE STATE MACHINE:
  draft
    → submitForReview()
    → pending_approval
       → reviewDraft() [pending_review | changes_requested]
       → editDraft() [creates new version, stays pending_approval]
       → rejectPlan() [back to draft]
    → approvePlan()
    → approved
       → publishPlan()
       → published (LOCKED - no edits allowed)
       → active (system assigned, next month)
       → closed (system assigned, month end)

BRANCHING:
  • At any non-published stage: editDraft() creates new version
  • At pending_approval: reviewDraft() can request changes
  • At pending_approval/changes_requested: rejectPlan() goes back to draft
  • At draft: rollbackToVersion() restores old version

STATISTICS CALCULATIONS
================================================================================

PLAN STATS:
  • totalAgencies    = count of assignments in current version
  • totalVisits      = same as totalAgencies
  • averagePerDay    = totalVisits ÷ 24 (working days)
  • workingDays      = 24 (baseline)
  • completionRate   = (totalVisits ÷ 72) × 100

MONTHLY EMPLOYEE STATS:
  • workingDays      = 24 baseline - employee holidays
  • targetVisits     = workingDays × 3 (daily quota)
  • completedVisits  = count of done VisitPlans that month
  • rate             = (completedVisits ÷ targetVisits) × 100
  • totalAgencies    = count of assigned agencies (active)
  • avgVisitPerDay   = completedVisits ÷ workingDays
